# PodForge HuggingFace Space 部署设计

**日期**: 2026-05-30
**状态**: 已批准
**负责人**: Claude + 用户

---

## 1. 目标

将 PodForge 完整部署为 HuggingFace Space，用户访问一个 URL 即可完成剧本输入 → 声线选择 → 生成播客 → 播放/下载的全流程，无需本地运行任何服务。

---

## 2. 架构

```
┌─────────────────────────────────────────┐
│       HuggingFace Space (T4 GPU)        │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  Gradio UI (简化版 6 组件)      │   │
│  │  - 剧本编辑器 (Textbox)         │   │
│  │  - 声线选择 (Dropdown)          │   │
│  │  - 情感标签 (Button group)      │   │
│  │  - 音频播放器 + 下载             │   │
│  │  - 示例剧本按钮 (4个)            │   │
│  └──────────────┬──────────────────┘   │
│                 │ 函数调用              │
│  ┌──────────────▼──────────────────┐   │
│  │  Python 逻辑层                 │   │
│  │  - drama_parser (已有)          │   │
│  │  - voice_manager (已有)         │   │
│  │  - VoxCPM2 TTS (按需加载)       │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

- **单 Space**：前端 UI + 后端逻辑 + TTS 模型，全部在 HF 免费 T4 GPU 上运行
- **按需加载模型**：第一次点"生成"时触发 VoxCPM2 加载，冷启动由用户自然承担
- **无持久化存储**：WAV 文件存内存，下载后即释放

## 5. 核心逻辑

### 模型加载策略（按需）
```python
model = None

def get_model():
    global model
    if model is None:
        model = VoxCPM.from_pretrained("openbmb/VoxCPM2", load_denoiser=False)
    return model
```

### 生成流程
```
用户提交剧本
  → drama_parser.parse_script()
    → voice_manager.assign_all()
      → get_model()  # 第一次触发模型加载
        → model.generate(text, cfg_value=2.0, inference_timesteps=10)
          → WAV numpy array
            → Gradio Audio(音频, sample_rate=48000)
```

### 错误处理
| 场景 | 处理 |
|------|------|
| 模型未加载时点生成 | 状态提示"正在加载模型..." → 完成后自动继续生成 |
| TTS 请求超时 | Gradio 返回错误文本，不崩溃 |
| 剧本格式错误 | drama_parser 抛出 ValueError，显示友好提示 |

## 7. 部署步骤（用户操作）

1. 注册 HuggingFace 账号 → 登录
2. 进入 https://huggingface.co/new-space → 选择 **Gradio** SDK → 选择 **T4 GPU** 硬件
3. 连接 GitHub 仓库（No1Hoo/PodForge）
4. HF 自动构建 → 部署完成
5. 用户打开 `https://用户名-podforge.hf.space` 即可使用
